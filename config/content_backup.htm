w10: Webots 是什麼？有什麼特點？ 如何進行場景模擬？ 如何用 Python 進行控制系統設計？ (Tutorial)

webots是什麼?

webots是一款由 Cyberbotics 開發、跨平台且專業的開源 3D 機器人模擬軟體。
它可用於研究、教育、快速原型製作，以及工業級機器人演算法開發。

你可以在其中建構 3D 世界、加入機器人、加入感測器與致動器，並透過程式語言（Python、C、C++、Java、ROS 等）控制它們的行為與決策。

webots的特點

1. 豐富的機器人與感測器庫

Webots 內建許多常見的機器人模型：

  • 車輪式機器人（如 Pioneer 3-DX, e-puck）

  • 四足/雙足機器人（如 Spot、NAO）

  • 機械手臂（UR5、KUKA…）

  • 飛行器（四旋翼、多旋翼）

也包含各式感測器：

  • 相機（Camera、RGB-D）

  • 雷射（Lidar）

  • IMU

  • GPS

  • 距離感測器、觸感感測器等。

2. 完整物理引擎支持

Webots 使用 ODE 物理引擎（並可切換其他引擎），提供：

  • 重力、摩擦、碰撞

  • 關節、平台、驅動馬達

  • 真實物理動力模擬
    適用於自走車、機械手臂、類人機器人、飛行器。

3. 可視化3D編輯器

可在 GUI 中輕鬆建立 3D 場景，包括：

  • 地形

  • 建築物

  • 障礙物

  • 道路、標誌

  • 光源、天空環境

拖拉方式建立場景十分直觀。

4. 跨語言控制

你可使用以下語言撰寫控制器：

  • Python（最常用）

  • C / C++

  • Java

  • MATLAB

  • ROS (Robot Operating System)

5. 與ROS完整整合

支援 ROS1、ROS2，可搭配：

  • Nav2

  • Slam Toolbox

  • RViz
    讓研究者能使用真實機器人軟體套件。

如何進行場景模擬？

Step1 - 建立或開啟一個世界(*.wbt)

  • 打開 Webots → File → New World

  • 或開現有範例，如：

    • robotics → mobile robots

    • manipulation → robot arms

Step2 - 在場景中加入物件

按右側 Add → Bot → Obstacle → Environment
可加入：

  • 地面（Floor / Terrain）

  • 桌子、箱子等障礙物

  • 機器人模型

  • 感測器（Lidar、Camera）

  • 光源、天空

Step3 - 加入控制器(controller)

每個機器人都必須綁定一個 controller 程式：

  1. 在 Robot 節點中找到 controller 欄位

  2. 選擇 new 建立 Python 控制器

  3. Webots 會建立 .py 檔並自動與機器人連結

Step4 - 執行模擬

  • 按 播放鍵 ▶

  • 控制器開始執行，機器人會依你的程式運作

  • 可隨時修改場景、調整物理參數、紀錄影片

如何用 Python 進行控制系統設計？

Webots 提供 Python API，以下示範基本流程：

Step1 - 匯入webots API

from controller import Robot

Step2 - 初始化機器人

robot = Robot()
timestep = int(robot.getBasicTimeStep())

Step3 - 取得馬達與感測器

例如控制輪式機器人:

left_motor = robot.getDevice('left wheel motor')
right_motor = robot.getDevice('right wheel motor')

left_motor.setPosition(float('inf'))  # velocity mode
right_motor.setPosition(float('inf'))

開啟距離感測器：

ds = robot.getDevice('ds0')
ds.enable(timestep)

Step4 - 主控制迴圈(控制系統核心)

while robot.step(timestep) != -1:
    distance = ds.getValue()

    # 簡單避障邏輯
    if distance < 800:
        left_motor.setVelocity(-2.0)
        right_motor.setVelocity(2.0)
    else:
        left_motor.setVelocity(4.0)
        right_motor.setVelocity(4.0)

w14: Webots 的模型有沒有辦法用程式方法建立? 有沒有辦法直接從 CAD 套件中轉出 Webots 的場景模型？

1️⃣ 建立檔案階段（程式生成）

  • 使用程式（Python、C++ 或腳本）生成 .proto 或 .wbt 檔案。

    • .proto：定義可重複使用的物件或機器人模型。

    • .wbt：定義整個世界（場景），可包含多個物件、光源、相機等。

  • 本質上都是 文字檔，語法類似 VRML / X3D。

2️⃣ Webots 啟動 & 解析階段

  1. 讀取檔案：Webots 將 .wbt 檔案文字內容讀入記憶體。

  2. 語法解析（Parser）：

    • Webots 內建 VRML/X3D 解析器，將文字描述轉換成 Scene Graph（場景圖）。

    • 每個節點（Solid、Shape、Light、Camera…）會生成 內部物件。

    • .proto 檔案會被展開成對應節點。

  3. 生成物理世界：

    • 使用 ODE 物理引擎計算剛體、摩擦、碰撞等。

    • 每個 Solid/物件生成物理剛體，套用質量、慣性、摩擦等屬性（如有設定）。

  4. 渲染與控制器初始化：

    • 場景進入 OpenGL 渲染引擎。

    • Controller（Python、C++ 等）初始化，開始控制機器人或物件。

    從 CAD 匯入模型

  • 支援格式：.x3d、.dae（COLLADA）、.stl、.obj。

  • CAD 匯出流程：

    1. 在 CAD 軟體完成模型（SolidWorks、Fusion360、Onshape）。

    2. 匯出 STL/OBJ/DAE。

    3. 放入 Webots 世界作為 Shape/Geometry。

  • 注意事項：

    • Webots 單位為 米，需確認 CAD 匯出單位。

    • STL/OBJ/DAE 主要是幾何形狀，物理屬性需在 Webots 設定。

    • 模型太複雜會影響運算效能，建議簡化網格。

  • 工具支援：

    • Onshape：可匯出 STL 或 OBJ。

    • Fusion360、SolidWorks：可匯出 COLLADA 或 STL。

利用基因演算法解決機械設計問題:

Length = 20

平面五連桿 Plotter 設計目標是針對一組 Length cm×Length cm 的繪圖區域，尋找並最佳化五連桿機構中的三個核心設計變數：連桿 L1、連桿 L2，以及底部距離 distance。最終目的是在確保機構穩定運行的前提下，找到能使連桿尺寸最小化（即材料成本與運動慣性最低）的設計。

一、幾何參數設定

    馬達間距 (LENGTH): 兩側連桿（L1 和 L4）的固定轉軸中心必須保持 Length cm 的水平間距。

    繪圖區域 (W×H): 機構必須能覆蓋一個 Length cm 寬度 (W) 和 Length cm 高度 (H) 的完整矩形區域。

    對稱性要求: 機構必須保持對稱，即 L1=L4 且 L2=L3。

二、最小化優化目標 (Objective)

設計應追求連桿總尺寸的最小化。優化目標 J 定義為：
最小化 J=1.0⋅L1+1.0⋅L2+0.5⋅distance

（此權重分配旨在優先最小化連桿長度 L1 和 L2，同時適度控制機構的底部高度 distance。）

三、運動學與工程約束 (Constraints)

設計結果必須嚴格滿足以下三個工程約束，以確保筆尖 C 點在整個繪圖區域內不會遇到死點或不可達的情況。

1. 可達性與外側奇異點迴避 (L1+L2 約束)

連桿 L1 與 L2 的長度總和必須大於機構在繪圖區域最遠點（即 X=Length,Y=distance+Length）處所需的最大距離 Dmax。

這項約束保證了筆尖在最遠處不會發生連桿 L1 和 L2 完全伸直共線（外側奇異點，即 A,B,C 共線）導致的鎖死。

2. 繪圖區域穩定性 (內側奇異點迴避)

連桿 L2 的兩倍長度 (2⋅L2) 必須大於筆尖 C 點在整個 Length cm×Length cm 繪圖區域內時，點 B 到點 D 之間的最大距離 Max(DBD)Area。

更重要的是，為了考慮實際製造公差和控制誤差，必須強制加入 1.0 cm 的安全裕度 (Δ)：
2⋅L2>Max(DBD)Area+1.0 cm

這項約束保證了筆尖在任何位置都不會發生連桿 L2 和 L3 完全伸直共線（內側奇異點，即 B,C,D 共線）導致的控制失效。

3. 轉軸方向約束 (Y 座標約束)

在求解逆運動學時，連桿 L1 和 L4 的擺動點 B 點和 D 點的垂直坐標 (YB 和 YD) 必須始終大於或等於零。

這確保了機構在整個工作過程中始終處於轉軸上方（上搖桿解）的穩定配置，防止機構翻轉至下方區域。

最小尺寸設計如下:

【基因演算法最終最小化尺寸設計結果】
找到**滿足約束且最小化尺寸**的設計（總尺寸 J ≈ 53.5 cm）:
L1 (Link 1 & 4) = 19.00 cm
L2 (Link 2 & 3) = 26.00 cm
Distance (Y-bottom) =17.00 cm
